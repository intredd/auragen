# Auragen — живые градиенты

> Интерактивный генератор анимированных WebGL-градиентов: настраиваешь сцену из
> «блобов», а результат забираешь как картинку, видео или встраиваемый
> веб-компонент. Инструмент для быстрой генерации фонов, постеров и визуалов под
> соцсети.

Живое демо: `https://<твой-логин>.github.io/auragen/`

---

## Что это и зачем

Auragen вырос из CodePen-демо с «живым» градиентом на сайте в полноценную
утилиту. Идея простая: собрать красивый анимированный градиент прямо в браузере,
управляя им мышью на канвасе, и одним нажатием получить готовый ассет —
изображение, короткое видео или кусок кода для вставки на свой сайт.

**Для кого:** дизайнеры и разработчики, которым нужен уникальный движущийся фон
для лендинга, поста или обложки без Photoshop/After Effects.

---

## Ключевые возможности

- **Сцена из блобов** — до 16 метаболов, у каждого свой цвет и параметры формы и
  деформации (никаких фиксированных 4 блобов и 2 общих цветов, как в исходной
  демо).
- **Прямое управление на канвасе** — перетаскивание якоря двигает блоб (на якоре
  виден его номер), перетаскивание пунктирного контура меняет размер, отдельная
  квадратная метка отвечает за поворот и «овальность». Помощники автоматически
  окрашиваются в контраст к фону.
- **Смешение цветов** — глобальный тумблер: блобы либо накладываются послойно,
  либо честно смешивают цвета в зонах пересечения.
- **Готовые демо-сцены** — меню с пресетами (дефолтная сцена и «Layered rings»),
  чтобы показать, что можно собрать.
- **Экспорт** — PNG и видео (WebM, запись по кнопке play/stop) в произвольном
  разрешении, независимом от размера окна.
- **Шаринг ссылкой** — вся сцена кодируется в URL (`?p=<base64>`), ссылку можно
  переслать или сохранить как пресет.
- **Встраивание** — фреймворк-независимый веб-компонент `<gradient-gen>` с той же
  картинкой, что в приложении.
- **Мелочи UX** — play/pause с заморозкой кадра, скрытие помощников, снятие
  выделения кликом по пустому месту, удаление блоба по `Delete`/`Backspace`.

## Технологии

- **React 18 + TypeScript**
- **Vite 6** (dev-сервер и сборка)
- **WebGL2 / GLSL ES 3.00** — рендер градиента
- **Zustand** — состояние (единый источник правды)
- Zero-dependency **Web Component** для встраивания

## Архитектура (и что здесь интересного инженерно)

Главный принцип: **движок градиента ничего не знает про React и UI.** Он
реализует небольшой интерфейс `GradientRenderer` (`setConfig`, `resize`, `play`,
`pause`, `renderFrame`…), поэтому WebGL-реализацию можно заменить на Canvas2D или
WebGPU, не трогая её потребителей — и ровно поэтому один и тот же движок работает
и в приложении, и во встраиваемом компоненте.

```text
src/
├─ app/                       # входная точка, корневой компонент, глобальные стили
├─ embed/gradient-gen.ts      # <gradient-gen> — веб-компонент (без React)
├─ features/
│  ├─ gradient/
│  │  ├─ engine/              # рендер-агностик ядро
│  │  │  ├─ types.ts          # интерфейс GradientRenderer
│  │  │  ├─ WebGLGradientRenderer.ts   # uniform-массивы, цикл до u_blob_count
│  │  │  ├─ shaders/          # вершинный + фрагментный шейдер
│  │  │  └─ index.ts          # createGradientRenderer()
│  │  ├─ config/              # createBlob / createDefaultConfig / demos / схемы полей
│  │  ├─ state/               # zustand-стор + engine context
│  │  ├─ components/          # GradientCanvas (мост React ⇄ движок)
│  │  ├─ utils/               # цвет + перевод координат экран ⇄ блоб
│  │  └─ types.ts             # Blob, GlobalSettings, GradientConfig
│  ├─ controls/               # UI управления, управляемый конфигом
│  ├─ export/                 # PNG/WebM, offscreen-рендер, сборка embed-сниппета
│  └─ presets/                # сериализация GradientConfig в/из URL
└─ main.tsx
```

Инженерные решения, которые стоит подсветить:

- **Config-driven UI.** Поля блоба и глобальные настройки описаны декларативно
  один раз (`blobFields`, `globalFields`) через `get`/`set`. Эти же схемы рисуют
  контролы и питают шейдер как uniform-массивы — добавление параметра не требует
  ручной проводки по слоям.
- **Единый контракт `GradientConfig`.** Одна структура обслуживает UI, шаринг по
  URL, экспорт медиа и встраиваемый компонент. Схема версионируется
  (`CONFIG_VERSION`), а десериализация всегда нормализует данные, поэтому старые
  ссылки не ломаются.
- **Рендер без ре-рендеров React.** `GradientCanvas` подписывается на стор
  *вне* React-рендера и толкает `setConfig` прямо в движок. Перетаскивание
  ползунка или блоба не вызывает ре-рендер канваса — только перерисовку на GPU.
- **Динамическое число блобов на GPU.** Параметры пакуются в
  предвыделенные типизированные массивы; фрагментный шейдер крутит цикл до
  `u_blob_count` (кап `MAX_BLOBS = 16`).
- **Экспорт в любом разрешении.** Рендер в offscreen-контекст, не зависящий от
  размера вьюпорта, плюс `preserveDrawingBuffer` для чтения кадра в PNG/видео.

### Модель данных

```ts
interface Blob {
  id: string;
  position: { x: number; y: number };  // uv-пространство, на экран как (x*W, y*H)
  size: number;                        // радиус в долях ширины
  scale: number;                       // «овальность» (растяжение по большой оси)
  angle: number;                       // поворот в градусах
  smoothStep: number;                  // резкость края
  cornerSmoothing: number;             // мягкость/растушёвка края
  deformationRatio: number;            // детализация «вобла»
  deformationSpeed: number;            // скорость дрейфа деформации
  separation: number;                  // амплитуда деформации
  color: string;                       // hex, у каждого блоба свой
}

interface GlobalSettings {
  backgroundColor: string;
  speed: number;       // множитель скорости анимации
  colorBlend: boolean; // смешивать цвета пересечений
}

interface GradientConfig {
  version: number;     // CONFIG_VERSION
  blobs: Blob[];
  global: GlobalSettings;
}
```

## Встраивание `<gradient-gen>`

Встраиваемая версия — это обычный кастомный HTML-элемент, работающий на том же
движке. Он не тянет React и подключается одним `<script>`.

### 1. Собрать компонент

```bash
npm run build:embed
```

На выходе — единый файл `dist-embed/gradient-gen.js`. Залей его на любой
статический хостинг (CDN, GitHub Pages, свой сервер).

### 2. Подключить на страницу

```html
<script src="https://your-host/gradient-gen.js" defer></script>

<gradient-gen style="display:block; width:100%; height:420px"></gradient-gen>
```

Без атрибутов элемент показывает дефолтную сцену. **Размер задаётся через CSS**
(`width`/`height`) — компонент подстраивает канвас под свой бокс через
`ResizeObserver`.

### 3. Вставить свою сцену

В приложении: **Export → Copy embed code**. В буфер попадёт готовый сниппет с
текущей сценой, «запечённой» в атрибут `preset`:

```html
<script src="https://your-host/gradient-gen.js" defer></script>
<gradient-gen
  preset="<base64url-config>"
  style="display:block; width:100%; height:420px"
></gradient-gen>
```

### Атрибуты

| Атрибут    | Значения                        | По умолчанию | Описание |
|------------|---------------------------------|--------------|----------|
| `preset`   | base64url-строка `GradientConfig` | дефолтная сцена | Кодированная сцена из «Copy embed code» / шаринга. |
| `autoplay` | `"false"` \| любое/отсутствует    | автозапуск   | `"false"` — стартует на паузе. |

Оба атрибута **реактивны**: меняешь их через JS — компонент подхватывает на лету.

```html
<gradient-gen id="hero" style="width:100%;height:50vh"></gradient-gen>
<script>
  const el = document.getElementById('hero');
  el.setAttribute('preset', '<новый-base64url-config>'); // сменить сцену
  el.setAttribute('autoplay', 'false');                  // поставить на паузу
</script>
```

### Локальная проверка

В корне репозитория есть `embed-example.html` — тестовая страница с двумя
компонентами. После `npm run build:embed` раздай корень репозитория статикой и
открой её.

> Требования: браузер с **WebGL2**. Для записи видео нужен **MediaRecorder API**
> (Chrome/Edge/Firefox; в Safari поддержка WebM ограничена).

## О чём рассказать на собеседовании

- Как декуплинг движка от UI позволил **переиспользовать один рендер** в React-
  приложении и во фреймворк-независимом веб-компоненте.
- Почему обновления идут в движок **в обход React-рендера** и что это даёт по
  перформансу при перетаскивании.
- Как **один конфиг** стал общим контрактом для UI, шаринга, экспорта и embed, и
  зачем ему версионирование + нормализация.
- Работа с **WebGL2**: динамическое число фигур через uniform-массивы, offscreen-
  рендер для экспорта в произвольном разрешении.
